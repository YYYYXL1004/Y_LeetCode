## 4.12
- 天梯赛 L2 专项：围绕“分数背包单位价值贪心 + 集合相似度双指针统计”完成两题，补齐了 set 交并统计与只求 size 的复杂度取舍
  - L2-003：月饼，分数背包（单位价值贪心）
    - 核心思路：先算单价 `dan = sale / cnt`，按单价降序，优先卖高单价品类，不足时卖部分
    - 模板要点：可拆分物品 => 贪心而非 DP，复杂度 `O(NlogN)`（排序主导）
    - 坑点：需求量 `D` 可能只覆盖最后一种的部分库存，收益要按 `dan * D` 累加
  - L2-005：集合相似度，set + 双指针统计交并大小
    - 核心思路：两个 set 天然有序，双指针线性统计交集个数 `Nc`，并集用 `|A| + |B| - Nc`
    - 模板要点：只求大小时不构造交并集容器，单次查询 `O(|A| + |B|)`
    - 坑点：输出百分号用 `%%`；并集不能直接写成两集合大小相加
- 详细笔记见 [贪心算法](../notes/贪心算法.md)、[STL速查表](../notes/STL速查表.md)

## 4.11
- 17 届蓝桥杯 C++A 组真题复盘：围绕“位运算构造、奇偶约束建模、答案二分、分段 DP、前缀和枚举”完成 A-G 七题回顾
  - P16224：均衡数，二进制构造 + 最近合法数比较
    - 核心思路：围绕目标数两侧分别构造最近的均衡候选，比较绝对差并处理平局
    - 模板要点：位运算构造高位/低位连续 1，避免枚举搜索
    - 坑点：高位左移必须使用 `1LL`，避免 32 位溢出
  - P16225：量子 2048，矩阵奇偶约束（GF(2)）+ 自由度计数
    - 核心思路：行列与 2x2 子块都是奇偶约束，化成异或方程后统计自由元
    - 模板要点：方案数 = `2^k mod MOD`，指数用快速幂
    - 坑点：奇偶方程推导必须在 mod 2 意义下进行
  - P16226：拦截程序，线段 minimax 中点结论
    - 核心思路：在两个可能位置间选整数点，最小化最坏距离，答案为 `ceil(|a-b|/2)`
    - 模板要点：先排序端点，再做奇偶上取整
    - 坑点：平移到整数点后要做上取整而不是四舍五入
  - P16227：切割木材，最小化最大值二分
    - 核心思路：二分挡板长度 L，判定总切割次数是否不超过 K
    - 模板要点：单根所需切割数为 `ceil(A_i/L)-1`
    - 坑点：二分边界和判定单调性要统一，防止死循环
  - P16228：读取指令，等差归一化 + 单段滑窗判定
    - 核心思路：先将目标按 C 归一化到 1..N 求和问题，先判 1 段是否可行
    - 模板要点：`W==0`、`W%C!=0`、`总和不足` 三个前置剪枝
    - 坑点：窗口扩张时注意右边界越界
  - P16229：外卖配送，批长代价预处理 + 分段 DP
    - 核心思路：先预处理一次送 L 单的最优代价，再做 `dp[i]` 枚举分割
    - 模板要点：新增批次固定代价 X 体现在转移里的 `+X`
    - 坑点：第一批没有折返代价，初值不能多加一次 X
  - P16230：综合应变指标，前缀和 + 三分点暴力枚举（复盘版）
    - 核心思路：前缀和 O(1) 求段和，三重循环枚举 `i<j<k`
    - 模板要点：用于小范围验证/对拍基准
    - 坑点：该写法 `O(N^3)`，只能过 60% 规模
- 详细笔记见 [动态规划](../notes/动态规划.md)

## 4.10
- 蓝桥杯冲刺专题：围绕“树上模板 + 区间贪心 + 翻硬币奇偶性”八题，强化了 DFS/LCA、区间调度与数学奇偶结论三类高频模型
  - 108：发现环，DFS 返祖边定位唯一环
    - 核心思路：图是“树 + 1 条边”，遇到访问过且不是父节点的点时，通过 fa 父链回溯得到整环
    - 模板要点：无向图 DFS 必须 `v == p` 过滤回边，发现环后立刻停止递归
    - 坑点：回溯收集环节点时别漏环起点；输出前记得排序
  - P3379：最近公共祖先（LCA），倍增 + DFS 预处理
    - 核心思路：先 DFS 预处理 `depth` 与 `fa[u][i]`，查询时先对齐深度，再同步上跳
    - 模板要点：`fa[u][i] = fa[fa[u][i-1]][i-1]`，单次查询 O(logN)
    - 坑点：深度对齐后先判 `u == v`；根由输入给定，预处理要从 S 出发
  - 2129：技能升级，优先队列贪心（阶段性）
    - 核心思路：每次选当前增益最大的技能升级，使用大根堆维护候选收益
    - 模板要点：正整数上取整次数用 `(a + b - 1) / b`，避免 `ceil(a / b)` 的整除陷阱
    - 坑点：该写法在 `M` 很大时会超时，后续需要补“阈值二分 + 等差求和”正解
  - 209：翻硬币，相邻翻转前缀纠错贪心
    - 核心思路：从左到右扫描，当前位置不一致就必须翻转 `i` 和 `i+1`
    - 模板要点：该类“局部只影响相邻位”的题，当前位置决策具有唯一性
    - 坑点：循环上界要到 `n-2`，防止访问 `i+1` 越界
  - 219：矩阵翻硬币，约数奇偶性结论
    - 核心思路：单格翻转次数是 `d(r)*d(c)`，奇偶只与是否完全平方数有关
    - 模板要点：`d(x)` 为奇数当且仅当 `x` 是完全平方数，答案转化为 `floor(sqrt(n))*floor(sqrt(m))`
    - 坑点：满分数据是超大整数；当前实现是 `long long` 考场部分分版本
  - P1803：线段覆盖，区间调度贪心
    - 核心思路：按右端点升序排序，能选就选，最大化不重叠区间数
    - 模板要点：端点相接可共存，判定条件是 `start >= ed`
    - 坑点：比较器与累计边界要统一，避免误删可选区间
  - 56：合并区间，排序后线性合并
    - 核心思路：按左端点排序，维护当前区间 `[st,ed]`，有交就扩展，无交就落盘
    - 模板要点：循环结束后要把最后维护的区间补进答案
    - 坑点：判重叠用 `intervals[i][0] <= ed`
  - 435：无重叠区间，最少删除数转最多保留数
    - 核心思路：按右端点升序保留最多不重叠区间，答案是 `n - 保留数`
    - 模板要点：这是 P1803 同模型的“对偶问法”
    - 坑点：`cnt` 统计的是保留数，不是删除数
- 详细笔记见 [图](../notes/图.md)、[贪心算法](../notes/贪心算法.md)、[位运算与数学](../notes/位运算与数学.md)

## 4.8
- 蓝桥杯模拟题串讲：子序列最值、斐波那契取模周期、反悔贪心三类模型
  - BS_3：删除 k 位得到最大数，单调栈贪心
    - 核心思路：从左到右维护“尽量大”的答案，遇到更大数字时弹出前面较小数字
    - 模板要点：遍历完若 `k` 未用完，需要从末尾继续删除
    - 坑点：单调栈不一定非要 `stack`，`string/vector` 用 `push_back/pop_back` 一样是栈语义
  - BS_4：斐波那契取模的第一个达标日，周期检测
    - 核心思路：状态由相邻两项决定，余数序列重复即进入周期
    - 判环方式：再次出现 `(1,1)` 说明进入下一周期，若此前没出现目标值则答案为 `-1`
    - 坑点：`a=1` 的最早达标日是第 1 天
  - BS_5：耐力闯关，排序 + 最大堆反悔贪心
    - 核心思路：按 `a+b` 升序处理（等价截止时间），每次先选，超约束就踢掉当前已选里 `a` 最大的关卡
    - 模板要点：这是“最多选任务”常见模板，`priority_queue` 维护可反悔集合
    - 坑点：累计和与比较器建议用 `long long` 防溢出
- 详细笔记见 [贪心算法](../notes/贪心算法.md)、[栈与队列](../notes/栈与队列.md)

## 4.5
- 回溯专题强化：把“组合去重”“排列去重”和“合法性约束剪枝”拆开理解，进一步区分不同题型该用的状态与剪枝条件
  - 22：括号生成，左右括号剩余量约束回溯
    - 状态设计：`left`、`right` 表示剩余左/右括号数量，`track` 表示当前构造结果
    - 关键剪枝：`right < left` 说明右括号数量已经不足以匹配当前前缀；`left < 0 || right < 0` 说明括号数量越界
    - 思维点：这类题不是“从数组里选数”，而是“当前位置能不能放这个字符”，重点在维护合法前缀
  - LCR 082：组合总和 II，排序 + 同层去重
    - 先排序，把重复元素放在一起，方便统一去重
    - 组合问题的去重发生在同一层：`i != start && nums[i] == nums[i - 1]` 直接跳过
    - 坑点：每个元素只能用一次，递归入口传 `i + 1`，不能传 `i`
  - LCR 084：全排列 II，`used` 标记 + 固定相同元素相对顺序
    - 排列问题没有 `start`，每层都从 0 开始枚举，只依赖 `used` 判断元素是否已使用
    - 去重条件：`nums[i] == nums[i - 1] && !used[i - 1]`，表示同层不能先选后面的重复元素
    - 核心区别：组合题关注“同层去重”，排列题关注“相同元素的相对位置约束”
- 详细笔记见 [回溯算法](../notes/回溯算法.md)
