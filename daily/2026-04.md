## 4.26
- 矩阵检索与子序列匹配专题：完成 4 道 LeetCode，形成了“一维化二分检索矩阵 + 右上角单调游走 + 单串双指针判定 + 多串位置索引二分统计”的组合模板
  - 74：搜索二维矩阵，一维化下标 + 二分查找
    - 核心思路：把 `m*n` 矩阵视作有序一维数组，`idx -> (idx/n, idx%n)` 映射回二维坐标后二分
    - 模板要点：只维护一套 `[0, m*n-1]` 边界，避免“先行后二次二分”的分支复杂度
    - 坑点：`mid` 映射时除数必须是列数 `n`，不是行数 `m`
  - 240：搜索二维矩阵 II，右上角双指针单调收缩
    - 核心思路：从右上角出发，当前值大于目标就左移，小于目标就下移，单调排除整行/整列
    - 模板要点：利用“行递增 + 列递增”构造可单调推进的路径，时间复杂度 `O(m+n)`
    - 坑点：起点选错会失去单调淘汰性质，推荐固定右上角或左下角
  - 392：判断子序列，双指针线性匹配
    - 核心思路：`i` 扫 `s`、`j` 扫 `t`，字符相等才推进 `i`，始终推进 `j`
    - 模板要点：单次判定最稳妥写法是“只回看是否 `i == s.size()`”
    - 坑点：循环条件应写成 `i < s.size() && j < t.size()`，避免越界
  - 792：匹配子序列的单词数，位置索引表 + `upper_bound`
    - 核心思路：预处理 `s` 中 26 个字符的所有出现位置；每个 `word` 逐字符二分找“严格大于上次位置”的最小下标
    - 模板要点：多串查询场景用“预处理一次 + 每串对数查询”比逐串双指针更高效
    - 坑点：必须用 `upper_bound`（找 `>`），若用 `lower_bound` 容易重复使用同一位置
- 详细笔记见 [双指针](../notes/双指针.md)

## 4.25
- 树上瓶颈路径（Maximin）专题：完成 1 道天梯 L2，重点是“路径唯一时用 DFS 传递路径最小边权，并在叶子中取最大瓶颈值”
  - 题型模板（可迁移）：
    - 适用信号：根到目标路径唯一，目标是最大化路径上的最小边权
    - 通用做法：设 `f[v] = min(f[parent], w(parent,v))`，遍历整棵树后在候选终点筛最大 `f`
    - 坑点：初值要高于边权上界；分支递归时不要污染兄弟分支状态
  - L2-059：森林藏宝图，树上 DFS 传递 bottleneck + 叶子筛选
    - 核心思路：按前驱关系建以 0 为根的树，DFS 过程中维护当前路径最小安全系数
    - 本题特化：终点是叶子节点；收集 `(叶子编号, 路径最小值)` 后取最优值并输出全部同值编号
    - 坑点：路径最小值更新应写成参数传递 `min(cur_min, edge)`，避免手动复原遗漏

- 图论最短路与图遍历综合专题：围绕“动态 Floyd（按时间逐步解锁中转点）离线查询、Dijkstra 的多目标维护（主目标最短/最省 + 次目标最大收益）、以及树/网格上的 BFS/DFS 染色统计”完成 7 题，并对 L2-001 的 Dijkstra 变形写法做了代码层面的复盘与注释完善（复盘不计入刷题数）
  - P1119：灾后重建，动态 Floyd（按时间解锁节点）
    - 核心思路：按村庄修复时间递增推进指针 `k`，每次只把新解锁的中转点加入 Floyd 的第 k 层松弛，回答任意时刻的两点最短路
    - 模板要点：查询是乱序的，但中转点解锁是单调的 → 用 `while(times[k] <= t)` 增量维护全源最短路
    - 坑点：若起点/终点在时刻 t 还未修复，必须直接输出 `-1`，不能只看 `dist`
  - L2-044：大众情人，Floyd 全源最短路 + “最差异性印象”最小化
    - 核心思路：先 Floyd 求任意两人间最短距离；对每个人 i 取所有异性 j 的 \(D_{ji}\) 最大值作为“最差印象”，再分别在男女中取该值最小者
    - 模板要点：先把“异性缘最好”转成 min-max 指标：\(\min_i \max_{j\in Opp(i)} D_{ji}\)
    - 坑点：方向是 **j 到 i**（别人对你），不是 i 到 j；Floyd 的 k 必须在最外层
  - L3-040：心情，Dijkstra（主目标最小花费）+ 次目标最大心情（同花费下取更大）
    - 核心思路：标准最短路松弛时维护两套值：`dist`（花费）与 `max_mood`（在最小花费约束下最大化心情）
    - 模板要点：多目标最短路常见写法是“先保证主目标单调最优，再在等价主目标里更新次目标”
    - 坑点：只在 `dist` 变小才入堆；`dist` 相等时更新次目标但不应破坏主目标的最短性
  - L2-023：图着色问题，边表校验 + 颜色集合计数
    - 核心思路：先统计颜色种类数是否等于 K，再逐边检查相邻点颜色是否冲突
    - 模板要点：点数大、查询多时，用边表 `edges` 逐条校验比邻接表更直接
    - 坑点：颜色必须“恰好使用 K 种”，不是“最多 K 种”
  - L2-031：深入虎穴，入度找根 + BFS 最后出队即最深处
    - 核心思路：用入度为 0 的点定位入口（根），对树做 BFS，最后一次弹出的节点就是最深处（按层序最靠后）
    - 模板要点：树结构题优先用“入度找根 + 层序遍历”稳定拿到深度/最后层信息
    - 坑点：输入是“每个点的孩子列表”，边是有向从父到子，不要反建错方向
  - L2-048：寻宝图，网格 DFS 连通块统计 + 岛内宝藏判定
    - 核心思路：遍历网格遇到陆地就 DFS 淹没该岛，岛内若出现 '2'~'9' 则记为“藏宝岛”
    - 模板要点：连通块统计统一用“遇到新块 +1，然后染色/淹没”
    - 坑点：宝藏判定要在淹没前读字符；DFS 边界与访问标记必须一致
  - L2-049：鱼与熊掌，倒排表（物品→拥有者）+ 双指针统计交集
    - 核心思路：每种物品存其拥有者有序表，查询 (x,y) 用双指针线性统计交集大小
    - 模板要点：交集只求数量时不建集合，`O(lenx+leny)` 直接数
    - 坑点：避免对向量做深拷贝；输入顺序天然有序时可省排序
- 详细笔记见 [最短路径](../notes/最短路径.md)（新增：动态 Floyd / 多目标最短路写法）、[图](../notes/图.md)、[岛屿问题](../notes/岛屿问题.md)

## 4.16
- 天梯赛树结构与栈结构综合专题：围绕“BST/镜像判定、完全树数组映射、父数组建树层深统计、树上路径字典序、在线中位数维护”完成 10 题，形成了“结构重建 + 遍历判定 + 数据结构拆分维护”的一体化解题框架
  - L2-004：这是二叉搜索树吗？，前序分段校验 + 递归后序构造
    - 核心思路：以前序首元素为根，按 BST（或镜像 BST）规则切分左右段并校验合法性，递归结束后按左-右-根输出后序
    - 模板要点：普通 BST 与镜像 BST 可共用一套递归骨架，只需整体翻转比较方向
    - 坑点：重复值归属要与题意一致（普通 BST 右子树可等于根）
  - L2-012：关于堆的判断，在线建小根堆 + 索引映射关系判定
    - 核心思路：插入时上滤建堆，额外维护 `key -> index`，把“兄弟/父子/根”判断转为下标关系
    - 模板要点：堆结构判断题本质是数组下标关系题，`i/2`、`2i` 是核心
    - 坑点：语句解析有多种句式，输入读取要完整消费无用词
  - L2-020：功夫传人，多叉树 DFS 传值累积
    - 核心思路：从祖师节点递归向下传播功力衰减率，到叶子（得道者）按权重累加贡献
    - 模板要点：树上“路径倍率”问题可统一为 DFS 参数携带当前贡献值
    - 坑点：最终输出要求向下取整，不能四舍五入
  - L2-026：小字辈，父数组建树 + BFS 最深层收集
    - 核心思路：先用父节点数组反建孩子表，再 BFS 维护深度并统计最大层
    - 模板要点：父数组题先找根再层序，最深层成员天然可一次遍历得到
    - 坑点：输出最深层节点时要保持编号递增
  - L2-035：完全二叉树的层序遍历，后序序列反填完全树数组
    - 核心思路：在下标空间按“左-右-根”递归访问，把后序序列顺次填入 `tree[u]`
    - 模板要点：完全二叉树优先用数组下标模拟结构，避免显式建指针树
    - 坑点：节点编号采用 1-indexed，边界是 `u > N`
  - L2-038：病毒溯源，树上 DFS 最长链 + 邻接表排序保字典序
    - 核心思路：枚举根到叶路径，维护当前路径与最优路径；仅在更长时更新答案
    - 模板要点：先排序邻接表可把“同长度字典序最小”转为遍历顺序保证
    - 坑点：更新条件要用 `>` 而非 `>=`，避免覆盖先出现的字典序最小路径
  - L2-045：堆宝塔，双栈（主结构+缓冲结构）规则模拟
    - 核心思路：A 维护当前塔的严格递减序列，B 暂存冲突元素，触发条件时先结算再回搬
    - 模板要点：在线模拟题可拆成“主结构 + 缓冲结构 + 触发结算条件”
    - 坑点：回搬时转移的是 B 的栈顶元素本身，不是当前输入值
  - L2-051：满树的遍历，父数组建树 + 满 m 叉树判定 + 前序输出
    - 核心思路：统计每个非叶节点孩子数，求最大分叉数 `k` 并检查是否所有非叶分叉一致
    - 模板要点：结构合法性判定与遍历输出可以共享同一份邻接表
    - 坑点：前序输出前需对子节点排序保证编号升序
  - L3-002：特殊堆栈，栈 + 树状数组维护中位数
    - 核心思路：栈负责 Push/Pop 顺序语义，树状数组维护值域频次；PeekMedian 用“前缀计数 >= k”的二分定位
    - 模板要点：顺序操作与统计查询分治到两种结构，可同时保证正确性与效率
    - 坑点：空栈下 Pop / PeekMedian 都要输出 `Invalid`
  - L3-010：是否完全二叉搜索树，反向 BST 建树 + 层序判完全性
    - 核心思路：按“左大右小”规则插入建树，层序时把空孩子也入队，用“先见空后见实”判定是否完全
    - 模板要点：完全二叉树判定可统一为层序遍历中的 `seen_null` 状态机
    - 坑点：层序判完全时不能跳过空节点，否则会误判

  - P1440：求m区间内的最小值，单调队列（历史窗口最小值）
    - 核心思路：在位置 r 查询区间 [r-m, r-1] 的最小值，维护按值单调递增的下标队列
    - 模板要点：队列存下标不存值，先入候选再弹过期，队头即当前答案
    - 坑点：本题区间不包含当前位置，入队元素应是 r-1
  - P1886：单调队列/滑动窗口，固定窗口最小值与最大值
    - 核心思路：最小值维护单调递增队列，最大值维护单调递减队列，窗口成形后读队头
    - 模板要点：每个下标最多入队出队一次，总复杂度 O(n)
    - 坑点：过期判定要和窗口左边界一致，常写作 `front <= r-k`
  - P1714：切蛋糕，前缀和 + 单调队列
    - 核心思路：固定右端点 r，在 k∈[r-m,r-1] 中找最小 preSum[k]，最大化 `preSum[r]-preSum[k]`
    - 模板要点：维护窗口内前缀和最小值队列，队头始终是当前最优被减数
    - 坑点：可能全负，答案初始化需为负无穷
  - P5788：单调栈，右侧第一个更大元素下标
    - 核心思路：维护按值单调不增的下标栈，当前值更大时持续弹栈并结算被弹位置答案
    - 模板要点：栈中存下标，结算语句是 `ans[st.top()] = i`
    - 坑点：题目要求严格大于，比较条件必须用 `>` 不能写成 `>=`
- 详细笔记见 [栈与队列](../notes/栈与队列.md)

  - 108：发现环，DFS 返祖边定位唯一环
    - 核心思路：图是“树 + 1 条边”，遇到访问过且不是父节点的点时，通过 fa 父链回溯得到整环
    - 模板要点：无向图 DFS 必须 `v == p` 过滤回边，发现环后立刻停止递归
    - 坑点：回溯收集环节点时别漏环起点；输出前记得排序
  - P3379：最近公共祖先（LCA），倍增 + DFS 预处理
    - 核心思路：先 DFS 预处理 `depth` 与 `fa[u][i]`，查询时先对齐深度，再同步上跳
    - 模板要点：`fa[u][i] = fa[fa[u][i-1]][i-1]`，单次查询 O(logN)
    - 坑点：深度对齐后先判 `u == v`；根由输入给定，预处理要从 S 出发
  - 2129：技能升级，优先队列贪心（阶段性）
    - 核心思路：每次选当前增益最大的技能升级，使用大根堆维护候选收益
    - 模板要点：正整数上取整次数用 `(a + b - 1) / b`，避免 `ceil(a / b)` 的整除陷阱
    - 坑点：该写法在 `M` 很大时会超时，后续需要补“阈值二分 + 等差求和”正解
  - 209：翻硬币，相邻翻转前缀纠错贪心
    - 核心思路：从左到右扫描，当前位置不一致就必须翻转 `i` 和 `i+1`
    - 模板要点：该类“局部只影响相邻位”的题，当前位置决策具有唯一性
    - 坑点：循环上界要到 `n-2`，防止访问 `i+1` 越界
  - 219：矩阵翻硬币，约数奇偶性结论
    - 核心思路：单格翻转次数是 `d(r)*d(c)`，奇偶只与是否完全平方数有关
    - 模板要点：`d(x)` 为奇数当且仅当 `x` 是完全平方数，答案转化为 `floor(sqrt(n))*floor(sqrt(m))`
    - 坑点：满分数据是超大整数；当前实现是 `long long` 考场部分分版本
  - P1803：线段覆盖，区间调度贪心
    - 核心思路：按右端点升序排序，能选就选，最大化不重叠区间数
    - 模板要点：端点相接可共存，判定条件是 `start >= ed`
    - 坑点：比较器与累计边界要统一，避免误删可选区间
  - 56：合并区间，排序后线性合并
    - 核心思路：按左端点排序，维护当前区间 `[st,ed]`，有交就扩展，无交就落盘
    - 模板要点：循环结束后要把最后维护的区间补进答案
    - 坑点：判重叠用 `intervals[i][0] <= ed`
  - 435：无重叠区间，最少删除数转最多保留数
    - 核心思路：按右端点升序保留最多不重叠区间，答案是 `n - 保留数`
    - 模板要点：这是 P1803 同模型的“对偶问法”
    - 坑点：`cnt` 统计的是保留数，不是删除数
