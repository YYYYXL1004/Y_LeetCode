# 滑动窗口

## 框架
```cpp
int left = 0, right = 0;
while (right < s.size()) {
    char c = s[right];
    right++;  // 扩大窗口
    // 更新窗口内数据...

    while (window needs shrink) {
        char d = s[left];
        left++;  // 缩小窗口
        // 更新窗口内数据...
    }
}
```

## 经典应用

### 最小覆盖子串
- 用 `need` 记录目标字符频率，`window` 记录窗口内字符频率
- `valid` 记录窗口中满足 `need` 条件的字符种类数
- 窗口收缩条件：`valid == need.size()`（满足覆盖要求时）
  ```cpp
  while(valid == need.size()) {
      if(right - left < len) { start = left; len = right - left; }
      char d = s[left]; left++;
      if(need.count(d)) {
          if(window[d] == need[d]) valid--;
          window[d]--;
      }
  }
  ```

### 字符串的排列 / 找所有字母异位词
- 固定窗口大小为目标串长度
- 当 `right - left >= n` 时判断是否满足条件并收缩窗口
  ```cpp
  while(right - left >= n) {
      if(valid == need.size()) return true; // 排列：返回true / 异位词：记录left
      char d = s[left]; left++;
      if(need.count(d)) {
          if(window[d] == need[d]) valid--;
          window[d]--;
      }
  }
  ```
- 当前 567 的实现也可写成最小覆盖同款模板：`while(valid == need.size())` 时判断 `right - left == n` 后收缩
- 两种写法本质等价：都是在长度约束下检查频次匹配，区别只在收缩触发条件
  ```cpp
  while(valid == need.size()) {
      if(right - left == n) return true;
      char d = s2[left++];
      if(need.count(d)) {
          if(window[d] == need[d]) valid--;
          window[d]--;
      }
  }
  ```

### 无重复字符的最长子串
- 窗口收缩条件：`window[c] > 1`（出现重复字符）
- 收缩后保证窗口内无重复，然后更新最大长度
  ```cpp
  while(window[c] > 1) { window[s[left]]--; left++; }
  res = max(res, right - left);
  ```
- 当前 3 的实现用 `unordered_map<char, int>` 维护字符频次，适合直接复用到 Unicode/更大字符集场景

### 将 x 减到 0 的最小操作数
- **转化思维**：从两端删除和为 x ⟺ 保留中间和为 `sum - x` 的最长子数组
- 窗口收缩条件：`windowSum > target`
- 答案为 `n - maxLen`（若不存在返回 -1）

### 乘积小于 K 的子数组（713）
- 题目是正整数数组，窗口乘积随右端扩张单调不减，可用双指针维护
- 收缩到 `windowProduct < k` 后，以 `right-1` 结尾的合法子数组个数为 `right - left`
  ```cpp
  int res = 0, left = 0;
  long long windowProduct = 1;
  for(int right = 0; right < nums.size(); right++) {
      windowProduct *= nums[right];
      while(left <= right && windowProduct >= k) {
          windowProduct /= nums[left++];
      }
      res += right - left + 1;
  }
  ```

### 最大连续1的个数 III（1004）
- 最多翻转 `k` 个 0 等价于窗口内 0 的数量不超过 `k`
- 可维护窗口内 1 的个数，条件写成：`窗口长度 - ones <= k`
  ```cpp
  while(right - left - ones > k) {
      if(nums[left] == 1) ones--;
      left++;
  }
  res = max(res, right - left);
  ```

### 替换后的最长重复字符（424）
- 设窗口内出现次数最多的字符频次为 `windowMax`
- 只要 `窗口长度 - windowMax <= k`，就能把其余字符替换成同一字符
- `windowMax` 在收缩时不必回退，保持历史最大值也能保证正确性

### 存在重复元素 II（219）
- 维护长度至多为 `k` 的滑动窗口 + `unordered_set`
- 每次加入新元素前先判重，若已存在则满足 `|i-j| <= k`

### 存在重复元素 III（220）
- 在 `|i-j| <= indexDiff` 的窗口内，还要满足 `|nums[i]-nums[j]| <= valueDiff`
- 用 `set<long>` 维护有序窗口，对当前值 `x`：
  - `lower_bound(x)` 检查最接近的较大值（ceiling）
  - 再检查前驱元素（floor）
- 只比较 floor/ceiling 两个最近邻即可，不需要遍历全窗口

## 滑动窗口 vs 双指针
- 滑动窗口适合"连续子数组/子串"问题
- 双指针更通用，可处理排序数组、链表等
