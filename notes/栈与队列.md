# 栈与队列

## 栈
- 核心特性：后进先出（LIFO），适合"最近匹配"或"回溯"问题
- 括号匹配：左括号入栈，遇到右括号时与栈顶匹配
- 路径简化：用栈维护目录层级，`..` 弹出栈顶，`.` 和空字符串跳过
- 用 `vector` 模拟栈：既有 `push_back/pop_back`，又方便最后顺序遍历
- `stringstream` + `getline(ss, token, '/')` 按分隔符分割字符串

## 单调栈（模板）

单调栈的本质是“维护一段满足单调性的历史决策”，不要求必须写 `stack<T>`。

- 可以用 `stack`，也可以用 `vector`、`string`、`deque`
- 只要你在尾部做 `push_back/pop_back`，就是栈语义
- 模板不止一种：从右到左“主动找答案”和从左到右“结算旧元素”都可以

典型写法（以“删除 k 位得到最大数”为例）：

```cpp
string ans;
for (char c : s) {
  while (!ans.empty() && k > 0 && ans.back() < c) {
    ans.pop_back();
    k--;
  }
  ans.push_back(c);
}
while (k-- > 0) ans.pop_back();
```

比赛常用模板（右侧第一个更大元素，左到右清算版）：

```cpp
vector<int> ans(n, -1);
vector<int> st; // 存下标，保持 a[st] 单调不增

for (int i = 0; i < n; i++) {
  while (!st.empty() && a[i] > a[st.back()]) {
    ans[st.back()] = a[i];
    st.pop_back();
  }
  st.push_back(i);
}
```

说明：上面不需要 `reverse`，答案在弹栈时就已经写到正确下标位置。

### P5788 单调栈（右侧第一个更大元素下标）

- 题目要的是下标，不是值，所以弹栈时结算 `ans[st.top()] = i`
- 栈维护按值单调不增，下标按时间递增，能保证“第一次被更大值击败”就是最近更大元素
- 比较符号必须严格用 `>`，因为题目要求的是“严格大于”

## 单调队列（模板）

单调队列适合处理“连续区间最值”问题，本质是维护一个按值单调的候选集合。

- 队列里存下标，不直接存值，便于判断元素是否过期
- 入队前弹出队尾中“不可能成为最优”的下标
- 再弹出队头中过期下标（超出窗口左边界）
- 队头即当前窗口最值对应下标

固定窗口最小值模板（窗口 `[r-k+1, r]`）：

```cpp
deque<int> dq; // 保持 nums[dq] 单调递增
for (int r = 0; r < n; r++) {
  while (!dq.empty() && nums[dq.back()] >= nums[r]) dq.pop_back();
  dq.push_back(r);
  if (!dq.empty() && dq.front() <= r - k) dq.pop_front();
  if (r >= k - 1) {
    int mn = nums[dq.front()];
  }
}
```

长度不超过 m 的最大子段和（P1714 模板化）：

```cpp
// preSum[i] = nums[1..i] 前缀和，固定 r 时在 k∈[r-m, r-1] 找最小 preSum[k]
deque<int> dq;
long long ans = LLONG_MIN;
for (int r = 1; r <= n; r++) {
  while (!dq.empty() && preSum[dq.back()] >= preSum[r - 1]) dq.pop_back();
  dq.push_back(r - 1);
  if (!dq.empty() && dq.front() < r - m) dq.pop_front();
  ans = max(ans, preSum[r] - preSum[dq.front()]);
}
```

常见坑点：
- 先后顺序错：通常先入新候选，再弹过期，再取答案
- 把过期条件写成值比较而不是下标比较
- 忘记“是否包含当前位置”的边界差异（如 P1440 取 `[r-m, r-1]`）

### BS_3 删除 k 位得到最大数

- 目标是保序前提下让结果尽量大
- 遇到更大数字时，尽量删除前面更小数字（且删除次数还剩）
- 扫描结束后若还可删，只能删末尾（此时后缀已无法通过交换改善）

补充：BS_4（斐波那契取模达标日）虽然也是从左到右线性扫，但核心是“周期检测”，不是单调栈模型。

## 受限栈/队列模拟（流程机模板）

当题目给出“多个容器 + 明确按键/操作规则 + 满载/空载特殊分支”时，优先用流程模拟而不是公式化推导。

- 常见容器映射：
  - 主输入流：`queue`
  - 临时缓存区：`stack`
  - 当前产物：`vector/string`
- 通用步骤：
  - 先把输入按题意建模到容器；
  - 维护主循环条件（只要还有可处理材料就继续）；
  - 按题目规定的优先级写 `if / else if` 分支；
  - 命中“结束当前段/当前件”条件后切段，继续下一段。
- 复杂度：元素通常只会有限次进出容器，常见是 `O(N)` 或 `O(N+Q)`。
- 通用坑点：
  - 分支顺序写错会改变语义（如“满载先弹再压”）；
  - 失败/结束分支若没处理好输入指针，后续状态会整体错位。

### 代表题

## 最小栈
- 辅助栈同步记录当前最小值，新元素 ≤ 栈顶时才入辅助栈
- 弹出时若等于最小值，辅助栈也要弹出
  ```cpp
  void push(int val) {
      stk.push(val);
      if(minStk.empty() || val <= minStk.top()) minStk.push(val);
  }
  void pop() {
      if(stk.top() == minStk.top()) minStk.pop();
      stk.pop();
  }
  ```

## 循环队列
- 使用 `capacity = k + 1`，浪费一个空间区分空和满
- 下一个位置：`(cur + 1) % capacity`
- 上一个位置：`(cur - 1 + capacity) % capacity`
- 判空：`front == rear`；判满：`(rear + 1) % capacity == front`
  ```cpp
  bool enQueue(int value) {
      if(isFull()) return false;
      data[rear] = value;
      rear = (rear + 1) % capacity;
      return true;
  }
  int Rear() {
      if(isEmpty()) return -1;
      return data[(rear - 1 + capacity) % capacity];
  }
  ```

## LRU 缓存
- 哈希表 O(1) 查找 + 双向链表 O(1) 增删
- 双向链表使用虚拟头尾节点简化边界处理
- 最近使用的放链表尾部，最久未使用的在头部
- 核心操作拆分：`makeRecently`、`addRecently`、`deleteKey`、`removeLeastRecently`
  ```cpp
  // 双向链表节点
  class Node {
  public:
      int key, val;
      Node *next, *prev;
  };
  // get: 不存在返回-1，存在则 makeRecently 后返回值
  // put: 已存在则删旧加新；不存在且满了则 removeLeastRecently 再添加
  ```
