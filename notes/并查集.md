# 并查集

## 核心概念

并查集（Union-Find）是一种用于管理元素分组的数据结构，支持两种核心操作：
- **合并（Union）**：将两个元素所在的集合合并
- **查找（Find）**：查询元素属于哪个集合（找根节点）

经典应用场景：
- 判断图的连通性
- 最小生成树（Kruskal 算法）
- 省份数量、岛屿问题等连通分量计数

## 优化版本（推荐记忆）

同时使用**路径压缩** + **按秩合并**，时间复杂度接近 $O(1)$（准确说是 $O(\alpha(n))$，$\alpha$ 是反阿克曼函数，增长极慢）。

```cpp
class UnionFind {
private:
    vector<int> parent;
    vector<int> rank;  // 秩：树的高度估计值
    int count;         // 连通分量个数

public:
    UnionFind(int n) {
        parent.resize(n);
        rank.resize(n, 1);  // 初始秩为 1
        count = n;
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }
    }

    // 路径压缩：查找时将节点直接连到根
    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);  // 递归压缩
        }
        return parent[x];
    }

    // 按秩合并：矮树挂到高树下
    void unite(int p, int q) {
        int rootP = find(p);
        int rootQ = find(q);
        if (rootP == rootQ) return;

        // 秩小的挂到秩大的下面
        if (rank[rootP] < rank[rootQ]) {
            parent[rootP] = rootQ;
        } else if (rank[rootP] > rank[rootQ]) {
            parent[rootQ] = rootP;
        } else {
            parent[rootQ] = rootP;
            rank[rootP]++;  // 秩相等时，合并后秩+1
        }
        count--;
    }

    bool connected(int p, int q) {
        return find(p) == find(q);
    }

    int getCount() {
        return count;
    }
};
```

## 两种优化详解

### 1. 路径压缩（Path Compression）

**目的**：降低树的高度，让查找更快

```cpp
// 递归版本（推荐，代码简洁）
int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);  // 顺带压缩
    }
    return parent[x];
}

// 迭代版本（避免栈溢出）
int find(int x) {
    int root = x;
    while (parent[root] != root) {
        root = parent[root];
    }
    // 第二遍遍历，将路径上所有节点直接连到根
    while (parent[x] != root) {
        int next = parent[x];
        parent[x] = root;
        x = next;
    }
    return root;
}
```

**效果**：查找后，路径上所有节点直接连到根，树变成扁平结构

```
查找前：          查找后：
    1               1
   / \            / | \
  2   3    →     2  3  4
 /
4
```

### 2. 按秩合并（Union by Rank）

**目的**：避免树退化成链表，保持树平衡

**秩（rank）**：树的高度的估计值（不是精确高度）

```cpp
void unite(int p, int q) {
    int rootP = find(p);
    int rootQ = find(q);
    if (rootP == rootQ) return;

    // 矮树挂到高树下面
    if (rank[rootP] < rank[rootQ]) {
        parent[rootP] = rootQ;
    } else if (rank[rootP] > rank[rootQ]) {
        parent[rootQ] = rootP;
    } else {
        parent[rootQ] = rootP;
        rank[rootP]++;  // 高度相等时，合并后高度+1
    }
    count--;
}
```

**为什么有效**：
- 总是把矮树挂在高树下面
- 树的高度增长缓慢，保持在 $O(\log n)$
- 结合路径压缩后，几乎接近 $O(1)$

## 时间复杂度对比

| 优化方式 | find | union |
|---------|------|-------|
| 无优化 | $O(n)$ 最坏 | $O(n)$ |
| 只路径压缩 | $O(\log n)$ 平均 | $O(\log n)$ |
| 只按秩合并 | $O(\log n)$ | $O(\log n)$ |
| **两种结合** | $O(\alpha(n))$ | $O(\alpha(n))$ |

> $\alpha(n)$ 是反阿克曼函数，对于实际应用中的 $n$（如 $n < 2^{65536}$），$\alpha(n) \leq 4$，可以视为常数。

## 常见变体

### 按大小合并（Union by Size）

用集合大小代替秩，有时更直观：

```cpp
vector<int> size;  // 记录集合大小

void unite(int p, int q) {
    int rootP = find(p);
    int rootQ = find(q);
    if (rootP == rootQ) return;

    // 小集合挂到大集合下面
    if (size[rootP] < size[rootQ]) {
        parent[rootP] = rootQ;
        size[rootQ] += size[rootP];
    } else {
        parent[rootQ] = rootP;
        size[rootP] += size[rootQ];
    }
    count--;
}
```

### 带权并查集

在边上记录额外信息（如距离、奇偶性等）：

```cpp
vector<int> parent;
vector<int> weight;  // 节点到父节点的权值

int find(int x) {
    if (parent[x] != x) {
        int root = find(parent[x]);
        weight[x] += weight[parent[x]];  // 路径压缩时更新权值
        parent[x] = root;
    }
    return parent[x];
}
```

## 使用示例

### 547. 省份数量

```cpp
int findCircleNum(vector<vector<int>>& isConnected) {
    int n = isConnected.size();
    UnionFind uf(n);

    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {  // 只遍历上三角
            if (isConnected[i][j] == 1) {
                uf.unite(i, j);
            }
        }
    }
    return uf.getCount();
}
```

### 684. 冗余连接

```cpp
vector<int> findRedundantConnection(vector<vector<int>>& edges) {
    int n = edges.size();
    UnionFind uf(n + 1);

    for (auto& edge : edges) {
        int u = edge[0], v = edge[1];
        if (uf.connected(u, v)) {
            return edge;  // 已经连通，这条边就是冗余的
        }
        uf.unite(u, v);
    }
    return {};
}
```

## 刷题心得

### 核心套路：先合并，后统计

并查集题目通常分三个阶段：

```
阶段1（输入）：合并 + 标记存在 + 记录个人属性
阶段2（分组）：遍历所有人，按 find(i) 分组
阶段3（输出）：统计计算 + 排序输出
```

**为什么这样做？**
- 实时维护成员列表很麻烦（合并时要搬运整个集合）
- 属性存在个人身上，不用担心根变了
- 最后统一处理，逻辑清晰

### 分组统计模板

```cpp
// 按根节点分组
map<int, vector<int>> families;
for(int i = 0; i < N; i++) {
    if(exist[i]) families[find(i)].push_back(i);
}

// 遍历每个分组统计
for(auto& [root, members] : families) {
    // 计算人均、排序等
}
```

### 天梯赛 vs LeetCode

| 平台 | 数据规模 | 优化要求 |
|------|---------|---------|
| LeetCode | 较大，有复杂度要求 | 路径压缩 + 按秩合并 |
| 天梯赛 | 较小（N ≤ 10^4） | 只用路径压缩即可 |

### 常见易错点

1. **忘记标记本人存在**：`exist[id] = true`（不仅是父母子女）
2. **根节点 ≠ 最小编号**：需要额外统计 `min_element`
3. **编号范围**：看清楚从 0 还是从 1 开始
4. **属性为 0 的人**：只作为亲属出现的人也要计入家庭
