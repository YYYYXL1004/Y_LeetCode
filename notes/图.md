# 图

## 1. 基础认知

### 1.1 图的表示

- 邻接表：适合稀疏图，常用写法是 `vector<vector<int>>` 或 `vector<vector<pair<int,int>>>`。
- 邻接矩阵：适合稠密图。

### 1.2 DFS 的本质

图遍历本质是多叉树遍历，加上状态避免重复访问。

```cpp
vector<bool> visited;
void traverse(vector<vector<int>>& graph, int s) {
    if (visited[s]) return;
    visited[s] = true;
    for (int next : graph[s]) {
        traverse(graph, next);
    }
}
```

## 2. 树上 DFS 模板（按你的代码风格）

这一节只提炼你当前代码风格，不做陌生化重构。

### 2.1 模板A：树上遍历（父节点过滤）

适用场景：树上统计、树上 DP、路径长度累积、找最远点。

```cpp
vector<vector<pair<int,int>>> mp;

void dfs(int u, int p, long long d) {
    // 前序：进入 u 时更新状态
    for (auto pa : mp[u]) {
        int v = pa.first, w = pa.second;
        if (v == p) continue;
        dfs(v, u, d + w);
    }
    // 后序：如有需要在这里汇总子树信息
}
```

固定检查点：
1. 状态放在哪（点、边、路径）。
2. 前序更新还是后序汇总。
3. 根节点初值、父节点过滤、`long long`。

### 2.2 模板B：关系判定（双 DFS）

适用场景：祖先链相交、两点关系判定、有限深度祖先查询。

```cpp
vector<int> mp[MAXN];
bool visited[MAXN];
bool has_common;

void dfs_u(int u, int gen) {
    if (gen > 5) return;
    visited[u] = true;
    for (int p : mp[u]) dfs_u(p, gen + 1);
}

void dfs_v(int v, int gen) {
    if (gen > 5 || has_common) return;
    if (visited[v]) {
        has_common = true;
        return;
    }
    for (int p : mp[v]) dfs_v(p, gen + 1);
}

bool check_relation(int u, int v) {
    fill(begin(visited), end(visited), false);
    has_common = false;
    dfs_u(u, 1);
    dfs_v(v, 1);
    return has_common;
}
```

固定检查点：
1. 每次查询前清空 `visited` 和 `has_common`。
2. 明确代数边界（本人算第几代）。
3. 剪枝条件先写好（深度超限/已命中）。

### 2.3 模板C：两次 DFS 求树直径

适用场景：目标函数可转化为“路径长度最大化”。

```cpp
vector<vector<pair<int, int>>> mp;
long long maxd = -1;
int max_node;

void dfs(int u, int p, long long d) {
    if (d > maxd) {
        maxd = d;
        max_node = u;
    }
    for (auto pa : mp[u]) {
        int v = pa.first, vd = pa.second;
        if (v != p) dfs(v, u, d + vd);
    }
}

long long get_diameter() {
    dfs(1, 0, 0);
    int a_node = max_node;
    maxd = -1;
    dfs(a_node, 0, 0);
    return maxd;
}
```

固定检查点：
1. 先证明最优值对路径长度单调。
2. 转化成“求直径 d”。
3. 把 d 代回原题公式。

## 3. 套题决策流程

1. 先判图类型：树 / DAG / 一般图 / 有向无向。
2. 如果是树，先看能否套模板A。
3. 如果是关系判定类，优先模板B。
4. 如果是最值且与路径长度单调，优先模板C。
5. 最后补题目个性约束（深度上限、权值公式、性别条件等）。

## 4. 题目映射（只写差异）

### 4.1 大臣的旅费（207）

核心映射：模板C。

这题费用函数是等差数列：

$$
11 + 12 + ... + (10 + d) = \frac{d(d+21)}{2}
$$

因为费用随 d 单调递增，所以先求树直径 d，再代公式。

坑点：
- 不要按边分段累加，先得到整段 d。
- 路径长度和费用建议用 `long long`。

### 4.2 愿天下有情人都是失散多年的兄妹（L2-016）

核心映射：模板B。

流程差异：
1. 先判同性，直接输出 `Never Mind`。
2. 异性再做五服内祖先相交判定。
3. 相交输出 `No`，不相交输出 `Yes`。

坑点：
- 本人是否算第 1 代要统一。
- 每次查询都要重置状态。
- 输入可能未显式给父母性别，需要补齐。

### 4.3 所有可能的路径（797）

核心映射：DAG 回溯。

差异点：
- DAG 无环，通常不需要全局 `visited`。
- 用 `path.push_back` / `path.pop_back` 维护当前路径。

### 4.4 克隆图（133）

核心映射：DFS + 哈希映射。

差异点：
- `originToClone` 维护原节点到新节点映射。
- 前序创建节点，递归后连接邻居。

### 4.5 发现环（108）

核心映射：树上 DFS + 返祖边判环。

差异点：
- 图结构是“树 + 1 条边”，因此只存在一个简单环。
- 当遇到 `visited[v] == 1 && v != p` 时，沿 `fa` 父链从当前点回溯到 v，即得到整环节点。
- 收集完环点后排序输出。

### 4.6 最近公共祖先（P3379）

核心映射：模板C 的祖先查询扩展（倍增 LCA）。

差异点：
- DFS 预处理 `depth[u]` 和 `fa[u][i]`。
- 查询时先把深点提升到同高度，再从高位到低位同步跳祖先。
- 本题根是输入 S，预处理从 S 开始。
