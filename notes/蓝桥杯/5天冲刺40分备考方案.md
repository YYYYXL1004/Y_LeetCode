# 蓝桥杯大学 A 组 C++ 省赛 5 天冲刺 40 分方案

## 一、目标

- 目标不是冲高分题，而是稳定拿到 `40` 分左右
- 核心思路是：
  - 前 `3` 题尽量全拿
  - 第 `4` 题争取拿到基础分或做出
  - 中档题不会时先写暴力和部分分版本，避免空题

## 二、基于当前仓库的判断

结合 [README](../../README.md)、[学习记录](../../daily/2025-03.md) 和现有题目代码，可以得出：

### 已有基础

- 模拟、字符串、哈希、前缀和、差分
- DFS、BFS、回溯
- 并查集、Dijkstra
- 基础 DP、二维 DP
- 位运算、筛法、快速幂

### 当前短板

- 真正的蓝桥杯 A 组题目刷得不多，比赛题感不足
- 题面风格切换不够熟，容易把“中档可拿分题”看成难题
- 树上高阶、数论高阶、数据结构高阶基本没有形成稳定战斗力

### 结论

- 你现在的主要问题不是“完全不会”，而是“比赛适配不够”
- 这 5 天不该补高难模型，而该把稳分题型练成条件反射

### 执行原则：真题主线 + 专题补洞

- 这份方案不是“只允许做下面列出的几道真题”，下面列出的题只是**每天的起步包和第一优先级**
- 主线依然是真题，因为你当前最缺的是：
  - 蓝桥杯题面适应
  - 比赛节奏
  - 止损能力
  - 部分分意识
- 但真题不是唯一题源。**只要能补当前短板，LeetCode 全站、历年蓝桥杯、洛谷、AcWing 都可以做**
- 这里尤其推荐 LeetCode，不是因为它“更像蓝桥杯”，而是因为：
  - 单一模型更纯，方便集中练
  - 题量大，适合快速补洞
  - 你仓库里已经积累了很多 LeetCode 题解和专题笔记，复盘成本更低
- 正确节奏不是“先把专题系统学完再刷真题”，而是：
  1. 先做当天真题
  2. 卡在某个模型上，就立刻切到同专题补 `2-5` 题
  3. 补完立刻回原真题重做
  4. 最后把结论整理进自己的模板页

### 先识别难度，再决定顺序

- 不要机械地按题号从 `A` 刷到 `H`
- 蓝桥杯大题虽然整体上通常“越往后越难”，但**并不是严格单调递增**
- 有些题题号靠后，但模型不重、实现不炸，反而是很好的冲分题
- 例如 `2025 F. 好串的数目`，就属于**题号偏后、但不一定比某些中前段题更难**的典型

建议每次拿到一整套题，先用 `5-10` 分钟做一次**扫题分层**：

**A 类：稳分题**

- 一眼能看出模型
- 实现量不大
- 边界和分类讨论可控
- 目标：优先拿下，保证得分下限

**B 类：高 ROI 中档题**

- 不是签到题，但模型相对明确
- 想一会儿能有思路
- 实现不算轻，但也不属于“重型工程”
- 目标：这是最该投入时间的题，往往决定你能不能从“只拿基础分”抬到 `40+`

**C 类：高风险题**

- 模型陌生，或者是明显重型树图 / 数论 / 数据结构
- 即使知道方向，实现量也很大
- 目标：短期冲刺阶段默认放后，不作为主线

你每次刷真题时，顺序应当是：

1. 先做 `A 类`
2. 再挑 `1-2` 道最有把握的 `B 类`
3. `C 类` 只在前面都打透、或者你明显已经具备模型基础时再碰

### 推荐优先级：不是按题号，而是按 ROI

下面这个推荐顺序更符合冲刺阶段的实际收益：

- **第一优先级**：稳分 `A 类`
  - 模拟、枚举、筛法、简单字符串、简单搜索、基础前缀和 / 差分
- **第二优先级**：高 ROI 的 `B 类`
  - 规律推导题
  - 按位贡献题
  - 中档 DP
  - 棋盘 / 回溯类搜索
  - 模型清楚但实现稍长的计数题
- **第三优先级**：高风险 `C 类`
  - 树上高阶
  - 重型数据结构
  - 很偏的数论
  - 综合性极强的树图题

这也是为什么：

- 不能把训练只困在“前 `4` 题”
- 也不能直接按年份整套硬刷到底
- 更合理的做法是：**先扫题分层，再优先做高 ROI 的题**

### 什么情况该切专题，什么情况不该切

**应该切专题补洞：**

- 某题想了 `20-30` 分钟还识别不出核心模型
- 看完题解后还是不能独立重写
- 同一类题连续卡住两次，说明不是单题问题，而是模型没掌握

**不应该切专题：**

- 你已经知道模型，只是实现细节写挂了
- 只是读题慢、分类讨论乱、边界没处理好
- 只是这一题题面恶心，但同类简单题你其实会做

这几种情况更应该做的是：**重写原题，而不是逃去刷新题。**

## 五、每天时间分配

如果你每天能拿出 `5h` 左右：

- `2.5h-3h` 限时做题
- `1.5h-2h` 复盘和重写
- `0.5h-1h` 只看自己的笔记

如果你每天只有 `3h-4h`：

- 保留“限时做题”
- 保留“重写错题”
- 砍掉额外拓展题

原则是：宁可少做题，也要把已做的题真正做会。

再强调一次：

- 真题是主线
- LeetCode / 其他题源是补洞工具
- 不要陷入“疯狂扩题”或“只刷自己舒服的题”的假努力
- 每次扩题都必须有明确目的：补当前真题暴露出来的短板

## 六、赛场策略

### 开场 10 分钟

- 先扫一遍所有题
- 优先标记：
  - 一眼模拟题
  - 一眼枚举题
  - 一眼筛法 / 位运算题
  - 一眼字符串题

### 前 90 分钟

- 目标是拿下最稳的 `3` 题
- 不要在第 `4` 题上卡太久

### 中段策略

- 某题超过 `25-30` 分钟还没出核心思路，立刻止损
- 能写暴力就先写暴力
- 能骗部分分就先骗部分分

### 最后 20 分钟

只检查这些：

- `long long`
- 数组越界
- 初始化是否清空
- 多组数据是否复位
- 边界输入
- 特判
- 输出格式
