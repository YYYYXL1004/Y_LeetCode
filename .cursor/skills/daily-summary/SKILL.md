# 每日学习总结

当用户要求"总结今天学习内容"时，按以下步骤执行。

## 强制约束（严格执行）

1. 必须先读本 SKILL，再开始任何总结编辑。
2. 每次总结都必须检查并同步三处：`daily/YYYY-MM.md`、`README.md`、`notes/专题.md`。
3. 若当天出现了可迁移的新题型/新坑点，必须更新至少一个对应专题笔记；不能只更新 daily 和 README。
4. 编辑代码或题解文件时，必须保留用户原有注释语义；不得无故删除用户注释。
5. 完成后必须做一致性自检并明确给出结论：
  - `README 当月增量 == daily 当日新增条目数 == README 分类新增条目数`。
6. 若任何一项无法同步完成，必须在回复中明确说明阻塞原因，不能跳过。

## 第一步：收集今日新增题目

1. 运行 git 命令获取今日新增和修改的文件：

```bash
# 获取今天新增的文件（未追踪 + 今日提交）
git status --porcelain
git log --since="today 00:00" --name-only --diff-filter=A --pretty=format:""
git diff --name-only HEAD
```

2. 筛选出 `LeetCode/`、`TianTi/`、`LanQiao/`、`Learn/` 目录下的 `.cpp` 文件
3. **阅读每个新增文件的代码**，提取：
   - 题号、题目名称
   - 难度等级
   - 使用的算法/数据结构
   - 关键思路和坑点
4. 对于非力扣题目（天梯赛、蓝桥杯等），根据代码中的题目注释识别来源和考察方向

### 题目来源识别规则

| 文件路径模式 | 来源 | 题号格式 |
|-------------|------|---------|
| `LeetCode/数字.xxx.cpp` | 力扣 | 数字 |
| `LeetCode/LCR xxx.cpp` | 力扣LCR | LCR xxx |
| `TianTi/L1/L1-xxx.cpp` | 天梯赛L1 | L1-xxx |
| `TianTi/L1/XB-L1-xxx.cpp` | 天梯赛校赛L1 | XBL1-xxx |
| `TianTi/L2/L2-xxx.cpp` | 天梯赛L2 | L2-xxx |
| `TianTi/L2/XB-L2-xxx.cpp` | 天梯赛校赛L2 | XBL2-xxx |
| `TianTi/L3/L3-xxx.cpp` | 天梯赛L3 | L3-xxx |
| `LanQiao/xxx.cpp` | 蓝桥杯 | 蓝桥杯题号 |

### 算法方向分类参照表

根据代码内容判断算法方向，常见分类：
- **动态规划**：dp数组、状态转移方程、备忘录递归
- **贪心算法**：局部最优选择
- **二分查找**：left/right边界、lower_bound/upper_bound
- **DFS/BFS**：队列/栈遍历、visited数组
- **回溯算法**：backtrack函数、选择/撤销
- **并查集**：find/union操作
- **最短路径**：Dijkstra、BFS
- **排序/归并/堆**：priority_queue、merge sort
- **双指针**：left/right或快慢指针
- **滑动窗口**：窗口收缩/扩展
- **位运算与数学**：异或、位移、取模
- **字符串**：string操作、KMP
- **链表**：指针操作
- **二叉树**：递归遍历、BST
- **图**：邻接表/矩阵
- **矩阵技巧**：螺旋遍历、旋转
- **前缀和与差分**：前缀和数组、差分数组
- **分治算法**：递归分解

## 第二步：更新每月学习记录

文件路径：`daily/YYYY-MM.md`（如 `daily/2025-03.md`）

### 格式要求

**插入位置**：在文件顶部（`# YYYY年M月 学习记录` 标题之后），在前一天的记录之前插入新的一天。日期按**降序排列**（最新在最前）。

**模板**（详见 [reference.md](reference.md)）：

```markdown
## M.DD
- 今日学习内容概要（归纳为专题方向）
  - 题号：题目名，核心算法
    - 关键思路点1
    - 关键思路点2
    - 坑点：需要注意的地方
- 详细笔记见 [专题名](../notes/专题名.md)（如有更新笔记）

|| 题号 | 题目 | 难度 | 状态 |
||------|------|------|------|
|| xxx | xxx | xxx | ✅ |

---
```

### 内容规范
- 概要部分要归纳出今日学习的**专题方向**，不是简单罗列
- 每题列出**核心算法**和**关键坑点**
- 天梯赛/蓝桥杯题目的难度写 L1/L2/L3 或对应等级
- 力扣题目难度写 简单/中等/困难
- 若当天题目可归并为同一题型，优先写“题型模板（通用套路）”，再写“各题差异点”，避免只做单题复述

## 第三步：更新专题笔记（仅限有代表性的题目）

路径：`notes/专题名.md`

### 判断标准：什么算"有代表性"
- 该专题的**新题型**或**新技巧**（不是已有套路的简单重复）
- 有独特的**思维转化**或**优化手段**
- 容易混淆或踩坑的题目

### 更新方式
1. 阅读现有笔记文件，理解已有内容
2. 在合适位置优先补“题型模板/万能框架”（先方法论，再题目）
3. 再补充新题差异化解析（边界、剪枝、公式、数据结构取舍）
4. 在笔记末尾的"相关题目"表格中添加新题（如果有这个表格）
5. 如果该笔记文件不存在，需新建（参考现有笔记格式）
6. 若判断“当天无代表性新题型”，必须在最终回复里写明“不更新 notes 的理由”；不能静默跳过。

### 笔记内容格式
```markdown
## 题型模板：xxx
**适用信号**：看到哪些条件可套此模板
**通用步骤**：1... 2... 3...
**复杂度**：...
**通用坑点**：...

## 题目差异：题目名（题号）
**相对模板的变化**：...
**本题特化处理**：...

### 方法一：xxx
代码块（带注释说明关键步骤）

### 方法二：xxx（如有）
代码块
```

## 第四步：更新 README.md

### 4.1 更新学习进度表

位置：`## 学习进度` 下方的表格

```markdown
| 月份 | 刷题数量 | 详情 |
|------|----------|------|
| 2025.03 | 数字+新增数 | [查看](./daily/2025-03.md) |
```

- 当月刷题数量 = 原数量 + 今日新增题目数
- **必须同步更新，不可省略**：凡是本次新增了 `LeetCode/`、`TianTi/`、`LanQiao/`、`Learn/` 下的题目或模板代码并已纳入README分类，都要同步更新当月刷题数量。
- 统计口径：与本次写入 `daily/YYYY-MM.md` 的条目数保持一致（包括竞赛题与学习模板条目），避免“分类表已更新但数量未更新”。
- 更新后执行一次一致性自检：`README 当月数量增量 == daily 当日新增条目数 == 本次新增到README分类表的条目数`。

### 4.2 更新题目分类表

位置：`## 题目分类` 下方对应分类的表格

- 将新题目添加到对应算法分类的表格中
- 每题一行，格式：`| 题号 | 题目名 | 难度 | 备注（核心算法/技巧） |`
- **非力扣题目也要按考察算法方向分类到对应表格中**
- 如果已有的分类中没有合适的，可以新建分类（参考已有分类的格式）
- 注意表格按题号排序

### 4.3 更新天梯赛/蓝桥杯备赛区域（如有新题）

位置：`## 天梯赛备赛` 下方对应分组的表格

- L1题目加到 `### L1 题目` 表格
- L2题目加到 `### L2 题目` 表格
- 格式：`| 题号 | 题目名 | [代码](./TianTi/Lx/文件名) |`

### 4.4 更新专题笔记表（如有新笔记文件）

位置：`## 专题笔记` 下方的表格

- 如果新建了笔记文件，需要在此表格中添加一行

## 执行顺序和检查清单

1. [ ] 运行 git 命令收集今日变更文件
2. [ ] 逐个阅读新增代码文件，提取题目信息
3. [ ] 更新 `daily/YYYY-MM.md`
4. [ ] 判断并更新 `notes/` 专题笔记（若不更新，需给出明确理由）
5. [ ] 更新 `README.md` 学习进度数字
6. [ ] 更新 `README.md` 题目分类表
7. [ ] 更新 `README.md` 天梯赛/蓝桥杯区域（如适用）
8. [ ] 最终确认所有文件一致性
9. [ ] 核对计数一致性：`README当月增量 = daily新增条目数 = README分类新增条目数`
10. [ ] 核对总结风格：已先写“可迁移模板”，再写“题目差异”

## 注意事项
- **不要删除任何已有内容**，只做增量添加
- 保持与现有格式完全一致（缩进、空行、表格对齐）
- 如果 `git log` 为空（今天没有提交），使用 `git status` 和 `git diff` 查看未提交的变更
- 非力扣题目（天梯赛/蓝桥杯）需要同时出现在：算法分类表 + 对应竞赛备赛区域
- 更新 README 时先定位到目标表格后再编辑，避免误插入到错误章节（例如把题目行插入“专题笔记”表）。
